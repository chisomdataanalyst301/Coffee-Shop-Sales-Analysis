# Coffee-Shop-Sales-Analysis
## Project Description
This project explores a coffee shop sales dataset to understand how the business performs over time. Using Microsoft Excel, the analysis focuses on uncovering patterns in revenue and customer transactions, identifying peak sales periods, and spotting opportunities to improve overall performance.
The work reflects a practical, real-world data analysis workflow, starting from messy, pivot-style data and transforming it into clear insights that a business owner or manager can actually use to make decisions.
It contain 149117 dataset.
## Project Overview
This project analyzes a coffee shop sales dataset using Microsoft Excel. The goal is to extract meaningful business insights from sales data, identify trends over time, and support decision-making with data-driven recommendations.
The dataset appears to be structured as a pivot-style Excel export, summarizing revenue and transaction counts across different months.
## Tools Used

Microsoft Excel (Primary tool)

Pivot Tables

Data Cleaning

Charts (Line, Column, Bar)
## Data Understanding
#### The dataset includes:

Monthly sales (Jan – Dec)

Revenue figures

Transaction counts

Aggregated metrics (likely from pivot tables)

#### However, the dataset required cleaning due to:

Unnamed columns

Mixed headers

Pivot-table formatting instead of raw tabular data
## Data Cleaning Process (Excel)
#### Steps performed:

1. Removed empty rows and irrelevant headers

2. Renamed columns properly (e.g., Month, Revenue, Transactions)

3. Converted text-based numbers into numeric format

4. Ensured consistent date/month formatting

5. Removed duplicate or repeated header rows
 ## Exploratory Data Analysis (EDA)
 #### Monthly Revenue Trend
 
Revenue generally increases over time

Noticeable growth from Q1 to Q2

Peak sales occur around mid-year

#### Transaction Volume

Transaction counts fluctuate across months

Some months show high transactions but relatively lower revenue

Suggests lower average order value

#### Revenue vs Transactions Insight

Not all high-transaction months produce the highest revenue

Indicates pricing, product mix, or upselling differences
## Key Insights
#### Seasonal Growth Pattern

Sales increase steadily after the early months

Likely influenced by customer behavior or promotions

#### Revenue Efficiency Matters

Some months generate more revenue with fewer transactions

Suggests higher-value purchases

#### Inconsistency in Sales Performance

Fluctuations indicate unstable demand or operational inconsistencies
## Visualizations (Created in Excel)

#### Recommended charts:

Line Chart → Monthly Revenue Trend

Column Chart → Monthly Transactions

Combo Chart → Revenue vs Transactions

These visuals help quickly identify trends and performance gaps.
## Business Recommendations
#### Focus on High-Performing Months

Replicate strategies used during peak months

#### Increase Average Order Value

Introduce bundles or upselling (e.g., coffee + pastry deals)

#### Stabilize Low Months

Run promotions during slow periods

#### Customer Behavior Analysis

Identify what drives higher spending months

## Limitations
Dataset is aggregated (pivot format)

No detailed product-level or customer-level data

Limits deeper analysis like customer segmentation
## Conclusion

This analysis demonstrates how Excel can be used to derive meaningful insights from sales data. Even with a pivot-style dataset, key business patterns such as seasonal trends, revenue efficiency, and performance gaps can be identified.
For future improvements, using raw transactional data would allow deeper insights such as customer segmentation, product performance, and predictive analytics.
## How to Reproduce (Excel Workflow)
Import dataset into Excel

Clean data (remove empty rows, fix headers)

#### Create pivot tables:

Rows → Month

Values → Revenue, Transactions

Insert charts for visualization

Analyze trends and document findings.
